    list 의 특징

    List인터페이스는 저장순서가 유지되고 중복을 허용하는 것이 특징이다.

    List 인터페이스에서 제공하는 메소드

    반환값 기능
    add(E e) boolean 리스트의 맨 끝에 요소를 추가합니다.
    addAll(Collection c) boolean 리스트의 맨 끝에 인자로 넘어온 리스트를 추가합니다.
    clear() void 리스트의 모든 요소를 제거합니다.
    contains(Object o) boolean 리스트에 인자로 넘어온 요소가 있다면 true를, 없다면 false를 반환합니다.
    containsAll(Collection c) boolean 인자로 넘어온 collection의 요소가 리스트안에 모두 있다면 true, 없다면 false를 반환합니다.
    get(int index) E 리스트의 index에 위치한 값을 반환한다.
    indexOf(Object o) int o에 해당하는 첫번째 요소의 인덱스를 반환합니다. 만일 찾고자하는 요소가없다면 -1을 반환합니다.
    iterator() Iteratir<E> 이 리스트에 해당하는 반복자를 반환합니다.
    remove(Object o) boolean o에 해당하는 첫번째 요소를 제거합니다.
    remove(int index) E 인덱스에 해당하는 요소를 제거합니다.
    removeAll(Collection c) boolean 컬렉션 c에 담긴 모든요소를 리스트에서 제거합니다.
    replaceAll(UnaryOperator operator) defalut void 리스트의 모든 요소를 operator로 연산한 결과로 대체합니다.
    retainAll(Collection C) boolean 컬렉션 c에 있지 않은 모든 요소를 제거합니다.
    set(int index, E element) E 인덱스의 요소를 인자로 넘어온 요소로 교체합니다.
    size() int 요소의 수를 반환합니다.
    sort(Comparator c) default void 리스트를 Comparator를 기준으로 정렬합니다.
    subList(int fromIndex, int toIndex) List fromIndex 부터 toIndex 까지의 리스트를 반환합니다.
    toArray() Object[] 리스트를 배열로 반환합니다.

    list 인터페이스를 구현한 클래스

    ArrayList
    ArrayList는 내부적으로 배열을 사용하여 요소를 저장합니다. 배열은 크기를 변경할 수 없는 인스턴스이므로 크기를 변경 하려고 할 때 새로운 배열을 만들어 저장합니다. 인덱스를 통해 요소에 빠르게 접근 할 수 있다는 장점이 있지만 크기의 제약이 있으므로 추가 제거할 때 시간이 많이 소요된다는 단점이 있습니다.
    LinkedList
    LinkedList는 각 노드간의 link를 가지는 구조를 갖고있습니다. 요소를 추가하거나 제거 할 때 노드간의 견결 link만 조작하면 되므로 추가 제거할 때 크기의 제약을 받지않습니다. 그러나 어떤 요소에 접근할 시 단번에 인덱싱을 통해 접근하지 못한다는 단점이 있습니다.
    Vector
    ArrayList와 같은 동작을 수행하는 클래스이지만, 기존코드와의 호환성을 위해서만 남아있습니다. Vector를 ArrayList가 대체합니다.

    간략한 예시

    name이라는 이름의 ArrayList를 생성해주었고 add메서드로 "Park","Jhon","Kali"를 넣어주었습니다. 출력결과를 통해 저장순서가 유지됨을 알 수 있습니다. remove메서드, contain메서드도 잘 동작합니다.
