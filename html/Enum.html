    • Enum

    enum은 관련성 있는 데이터를 상수처럼 정의해서 사용하는 문법입니다.
    상수는 프로그램상에서 해당 값의 의미를 명확히 하기 위해 사용합니다. 배열의 길이를 MAX_LENGTH로 지정해서 사용하기도 하고, MATH api 에서는 PI값을 상수로 지정해 놓기도 합니다. 상수는 값에게 의미를 부여하는 역할을 합니다.
    그러나 때때로 값보다 존재 자체로 필요한 상수들이 있습니다.

    클래스 Cafe는 네개의 상수를 갖고있습니다. 1은 스타벅스에, 2는 이디야, 3은 커피베이, 4는 할리스 라는 이름에 매핑되어 있습니다.
    저 카페 이름들과 숫자와의 연관성은 전혀 없다고 볼 수 있습니다. STARBUCKS 라는 이름을 쓰기 위해 1이라는 연관성 없는 숫자를 끌어와서 쓴 것 입니다.
    main함수 에서의 cafe 멤버 변수의 호출은 이와 같은 상수 선언의 문제점을 보여주고 있습니다.
    cafe멤버 변수를 출력하니 얻고 싶은 카페이름(STARBUCKS, HOLLIS, etc)이 아닌 숫자를 반환합니다. 코드를 사용하는 사용자의 입장에서는 Cafe 클래스를 직접 열어서 해당 숫자가 어떤 카페의 이름과 매핑되어 있는지 확인하는 불편한 절차를 거치지 않는 이상 그 숫자의 의미를 제대로 파악 할 수 없습니다.

    Enum의 사용

    enum의 사용 방법은 다음과 같습니다. 

    {접근 제한자} enum enumName{
    값1, 값2, 값3, ...
}

    cafe 변수의 타입을 CafeType으로 지정하였고 생성자에서 매개변수의 타입을 CafeType으로 지정하였기 때문에 인자 값은 CafeType만 전송 할 수 있고 일반 상수 값은 사용할 수 없습니다.
    Cafe클래스의 인스턴스가 어떤 카페인지 출력할 때 더이상 의미를 알 수 없는 숫자값이 출력되지 않고 카페 이름이 출력되는 것 도 확인 할 수 있습니다.

    ordinal(), values()

    enum에는 몇가지 유용한 메소드를 제공 합니다.

    values() : 해당 enum에 저장되어 있는 값들을 해당 enum 타입의 배열로 반환해 준다.
ordinal() : enum 안에 정의되어 있는 각 값들의 인덱스 번호를 반환해 준다.
